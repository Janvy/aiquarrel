## REQ-001: [图标资源] TabBar 底部导航图标

- **状态**: ✅ 已完成
- **提出时间**: 2026-05-28
- **目标 Agent**: `ui.md`
- **关联模块**: `app.config.ts` TabBar 配置
- **优先级**: P0（阻塞打包审核 — 微信小程序要求 TabBar iconPath 必须指向真实存在的本地图片，否则无法通过审核）

### 需求描述

`app.config.ts` 中 TabBar 配置引用了 6 个图标文件，但 `frontend/src/assets/icons/` 目录不存在，所有图标文件均缺失。微信小程序规范要求 TabBar 图标必须是真实存在的本地 PNG 文件，不能使用 emoji 或网络图片替代。

缺失的 6 个图标文件：

| 文件名 | 用途 | 场景 |
|--------|------|------|
| `home.png` | 首页图标 | Tab 未选中态 |
| `home-active.png` | 首页图标 | Tab 选中态（主色 #FF6B6B） |
| `history.png` | 历史图标 | Tab 未选中态 |
| `history-active.png` | 历史图标 | Tab 选中态（主色 #FF6B6B） |
| `mine.png` | 我的图标 | Tab 未选中态 |
| `mine-active.png` | 我的图标 | Tab 选中态（主色 #FF6B6B） |

### 上下文信息

- **相关文件**: `frontend/src/app.config.ts` 第 24-37 行
- **图标路径约定**: 所有图标放在 `frontend/src/assets/icons/` 目录下
- **技术约束**:
  - 微信小程序要求：TabBar 图标必须是本地图片，大小限制 40KB 以内
  - 建议尺寸：81x81px（@1x），建议同时提供 @2x（162x162px）和 @3x（243x243px）
  - 格式：PNG（支持透明背景）
  - 未选中态使用 `#999999` 灰色调，选中态使用 `#FF6B6B` 主色调
- **设计风格**: 年轻、轻松、情绪化。当前项目使用 emoji 作为图标语言：首页 🏠、历史 📋、我的 👤，图标设计风格可参考这些 emoji 的语义

### 期望交付

1. 在 `frontend/src/assets/icons/` 目录下放置上述 6 个 PNG 图标文件
2. 图标风格与项目整体调性一致（年轻、轻松、适合微信传播）
3. 在 `ui.md` 的响应中说明图标的设计语义和配色方案

### 响应

**响应时间**: 2026-05-28 | **响应 Agent**: `ui.md`

**设计规格**:

| 文件名 | 用途 | 尺寸 | 颜色 |
|--------|------|------|------|
| `home.png` | 首页未选中 | 81×81px | `#999999` |
| `home-active.png` | 首页选中 | 81×81px | `#FF6B6B` |
| `history.png` | 历史未选中 | 81×81px | `#999999` |
| `history-active.png` | 历史选中 | 81×81px | `#FF6B6B` |
| `mine.png` | 我的未选中 | 81×81px | `#999999` |
| `mine-active.png` | 我的选中 | 81×81px | `#FF6B6B` |

**图标设计语义**:

| Tab | 参考 emoji | 图形描述 | 设计要点 |
|-----|-----------|----------|----------|
| 首页 | 🏠 | 尖顶房屋轮廓 | 三角形屋顶 + 矩形屋体 |
| 历史 | 📋 | 表盘 + 列表横线 | 圆形表盘（右上指针）+ 三条横线 |
| 我的 | 👤 | 人物轮廓 | 圆形头部 + 弧形身体轮廓 |

**视觉风格**: 线性图标，2px 描边，圆角端点，参考 Feather Icons 风格。格式 PNG-8，透明背景，单文件 < 5KB。

**获取方式**: 推荐在 Figma 中使用 Feather Icons 的 `home`/`clock`/`user` 图标，导出两个颜色版本（`#999999` 未选中，`#FF6B6B` 选中），共 6 个文件放入 `frontend/src/assets/icons/`。

## REQ-002: [插画/图片资源] 空状态插画

- **状态**: ✅ 已完成
- **提出时间**: 2026-05-28
- **目标 Agent**: `ui.md`
- **关联模块**: `components/EmptyState/` 组件
- **优先级**: P1（影响体验 — 当前使用 emoji 💬 作为占位，可上线但视觉效果较弱）

### 需求描述

`EmptyState` 组件目前使用 emoji `💬` 作为空状态插画占位符。该组件在以下场景被使用：

| 页面 | 触发条件 | 当前展示 |
|------|----------|----------|
| 历史页 | 用户没有生成过任何话术 | 💬 + "还没有生成过怼人话术，去首页试试吧" |
| 收藏页 | 用户没有收藏过任何话术 | 💬 + "还没有收藏，去生成喜欢的怼人话术吧" |

需要使用更有视觉吸引力的插画替代 emoji 占位符。

### 上下文信息

- **相关文件**: `frontend/src/components/EmptyState/index.tsx`
- **技术约束**:
  - 微信小程序包体积限制（总包 2MB），插画资源需控制在合理大小（建议单张 < 50KB）
  - 支持 rpx 自适应，插画展示区域约 120rpx × 120rpx
  - 格式建议：PNG（带透明背景）或 SVG（通过 image 组件加载）
- **设计风格**: 轻松、可爱、贴合"情绪表达"主题。可参考对话气泡、写字、思考等意象
- **建议**: 1 张通用空状态插画即可，两个场景共用

### 期望交付

1. 在 `frontend/src/assets/images/` 目录下放置空状态插画文件（建议命名为 `empty-state.png`）
2. 如有不同场景需要不同插画，分别提供并说明对应关系

### 响应

**响应时间**: 2026-05-28 | **响应 Agent**: `ui.md`

**设计规格**:

| 属性 | 值 |
|------|-----|
| 文件名 | `empty-state.png` |
| 路径 | `frontend/src/assets/images/empty-state.png` |
| 展示尺寸 | 240×160rpx |
| 实际尺寸 | 480×320px (@2x) |
| 格式 | PNG-24，透明背景 |
| 最大体积 | < 50KB |

**视觉描述**:

```
主视觉元素:
  - 一个大的对话气泡（圆角矩形 + 左下小三角），边框色 #FF6B6B，填充色 #FFF0F0
  - 气泡内有三个小圆点（省略号 ···），颜色 #FF6B6B
  - 气泡右上角有一颗小星星 ✨（黄色 #FFD93D）
  - 气泡下方有几个更小的淡色气泡或逗号形状装饰

整体构图:
  居中排列，插画在上，引导文案在下
  插画区域约 240rpx 正方形
  文案区域约 120rpx 高度
```

**使用场景**:

| 页面 | 触发条件 | 展示文案 |
|------|----------|----------|
| 历史页 | 无生成记录 | "还没有生成过怼人话术\n去首页试试吧" |
| 收藏页 | 无收藏记录 | "还没有收藏\n去生成喜欢的怼人话术吧" |

两个场景共用一张插画，文案通过 EmptyState 组件的 props 区分。

## REQ-003: [UI设计] 分享卡片设计模板

- **状态**: ✅ 已完成
- **提出时间**: 2026-05-28
- **目标 Agent**: `ui.md`
- **关联模块**: `pages/result/` 分享图片功能
- **优先级**: P2（锦上添花 — 可通过 Canvas 绘制，但需要设计稿指导排版）

### 需求描述

结果页有"生成分享图片"功能。当前实现调用后端 API-08 `POST /api/v1/share-image`（备选方案：前端 Canvas 绘制）。无论哪种方案，都需要一个分享卡片的视觉设计模板来确定：
- 分享图片的整体尺寸和比例
- 文案展示区域的排版（字号、行距、对齐方式）
- 风格标签、项目名称、二维码/水印等元素的位置
- 背景色/渐变/装饰元素

### 上下文信息

- **相关页面**: `frontend/src/pages/result/result.tsx` 分享功能
- **技术约束**:
  - 微信小程序分享图片建议尺寸：5:4 比例（如 500x400px 或 750x600px）
  - 如使用前端 Canvas，需转换为 Canvas 绘制指令
  - 如使用后端生成，需提供 PNG 格式的设计稿
- **内容元素**: 生成的怼人文案、风格标签（emoji + 名称）、项目名称"AI吵架生成器"、可能的二维码引导关注
- **设计风格**: 与项目整体调性一致，适合在朋友圈/群聊中传播，具有视觉吸引力

### 期望交付

1. 分享卡片的视觉设计稿（ASCII 线框图 + 标注各元素的尺寸/位置/颜色）
2. 建议的配色方案（背景色、文字色、装饰元素色）
3. 如果可能，提供一张设计参考图
