# AI吵架生成器 — 页面需求

## 1. 页面结构总览

```
├── app.json (底部 TabBar)
│   ├── Tab1: 首页 (pages/index)
│   ├── Tab2: 历史 (pages/history)
│   └── Tab3: 我的 (pages/mine)
│
└── 子页面
    └── 结果页 (pages/result)
```

## 2. 首页 (pages/index)

### 2.1 页面布局

```
┌─────────────────────────────┐
│  🎯 AI吵架生成器            │  ← 顶部标题栏
├─────────────────────────────┤
│                             │
│  ┌───────────────────────┐  │
│  │                       │  │
│  │   输入你想怼的场景...    │  │  ← 文本输入区
│  │                       │  │     (textarea, 最多200字)
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  ┌────────────────────────┐ │
│  │ 同事甩锅 │对象冷暴力 │..│ │  ← 预设场景标签
│  └────────────────────────┘ │     (横向滚动)
│                             │
│  选择风格：                  │
│  ┌──────┬──────┬──────┐    │
│  │ 🤝  │ 🎭  │ 🤪  │    │
│  │高情商│阴阳  │发疯  │    │  ← 风格选择卡片
│  ├──────┼──────┼──────┤    │     3列网格布局
│  │ 📝  │ 🕶️   │      │    │     选中态：边框高亮+缩放
│  │文艺  │霸总  │      │    │
│  └──────┴──────┴──────┘    │
│                             │
│  ┌───────────────────────┐  │
│  │     生成怼人话 😈      │  │  ← 生成按钮
│  └───────────────────────┘  │     (主色大按钮，圆角)
│                             │
└─────────────────────────────┘
```

### 2.2 交互说明

| 元素 | 交互 | 说明 |
|------|------|------|
| 文本输入框 | 点击输入 | 占位文案：输入你想怼的场景... 最多200字 |
| 预设场景标签 | 点击填充 | 点击后自动填入输入框 |
| 风格选择卡片 | 单选 | 默认选中"发疯文学版"；每个卡片有 emoji + 名称 |
| 生成按钮 | 点击 | 输入为空时置灰不可点；点击后显示 loading，然后跳转结果页 |

### 2.3 状态说明

| 状态 | 触发条件 | 表现 |
|------|----------|------|
| 默认态 | 刚进入页面 | 输入框为空，风格默认发疯文学，按钮可点（允许自由生成） |
| 输入中 | 用户输入文字 | 字数统计实时更新 |
| 生成中 | 点击生成按钮 | 按钮变为 loading 态，文案"正在酝酿怼人话术..." |
| 完成 | 生成完毕 | 自动跳转结果页 |
| 错误 | 生成失败 | Toast 提示"生成失败，请稍后再试" |

## 3. 结果页 (pages/result)

### 3.1 页面布局

```
┌─────────────────────────────┐
│  ← 返回      怼人话术       │  ← 导航栏
├─────────────────────────────┤
│                             │
│  ┌──┬──┬──┬──┬──┐          │
│  │高│阴│发│文│霸│          │  ← 风格Tab切换
│  │情│阳│疯│艺│总│          │     当前风格高亮
│  └──┴──┴──┴──┴──┘          │
│                             │
│  ┌───────────────────────┐  │
│  │                       │  │
│  │    [生成的文案内容]    │  │  ← 结果展示区
│  │                       │  │     卡片样式
│  │                       │  │     字号较大，行间距适中
│  │                       │  │
│  └───────────────────────┘  │
│                             │
│  ┌────┐ ┌────┐ ┌────┐     │
│  │复制│ │收藏│ │再来│     │  ← 操作按钮行
│  └────┘ └────┘ └────┘     │
│                             │
│  ┌───────────────────────┐  │
│  │     生成分享图片       │  │  ← 次要操作
│  └───────────────────────┘  │
│                             │
└─────────────────────────────┘
```

### 3.2 交互说明

| 元素 | 交互 | 说明 |
|------|------|------|
| 风格Tab | 点击切换 | 切换后重新请求同场景不同风格；当前Tab高亮，内容区刷新 |
| 结果卡片 | 静态展示 | 显示生成文案 |
| 复制按钮 | 点击 | 复制全文到剪贴板，Toast "已复制" |
| 收藏按钮 | 点击 | 切换收藏/取消收藏状态，图标和颜色变化 |
| 再来一句 | 点击 | 同场景同风格重新生成，loading 后更新内容 |
| 生成分享图片 | 点击 | 调起 Canvas 生成卡片图，弹出保存引导 |
| 返回按钮 | 点击 | 回到首页（保留之前的输入和风格选择） |

### 3.3 状态说明

| 状态 | 表现 |
|------|------|
| 加载中 | 结果卡片区显示骨架屏 |
| 有内容 | 展示文案 + 操作按钮 |
| 切换风格中 | 卡片区 loading，不跳页面 |
| 复制成功 | Toast "已复制到剪贴板" |
| 生成图片中 | 全屏 loading "正在生成图片..." |
| 收藏/取消收藏 | 图标即时切换 |

## 6. 通用规范

### 6.1 底部 TabBar

| Tab | 图标 | 文案 | 页面路径 |
|-----|------|------|----------|
| 首页 | 🏠 | 首页 | pages/index |
| 历史 | 📋 | 历史 | pages/history |
| 我的 | 👤 | 我的 | pages/mine |

### 6.2 全局 Loading

- 使用微信原生 `wx.showLoading()`
- 文案："正在酝酿怼人话术..."

### 6.3 全局 Toast

- 成功：绿色对勾 + 文案（如"已复制"）
- 失败：红色感叹号 + 文案（如"生成失败"）
- 使用微信原生 `wx.showToast()`

### 6.4 配色方案

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | #FF6B6B | 活力红橙，按钮、选中态 |
| 辅助色 | #FFD93D | 暖黄，高亮、收藏 |
| 背景色 | #F8F9FA | 浅灰背景 |
| 卡片色 | #FFFFFF | 白色卡片 |
| 文字主色 | #333333 | 正文 |
| 文字辅色 | #999999 | 次要信息 |
