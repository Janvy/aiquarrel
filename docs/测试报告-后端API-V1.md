## 4. 缺陷详情

### BUG-001：无效风格参数返回错误码不正确

- **严重程度**: 一般
- **接口**: `POST /api/v1/generate`
- **复现步骤**: 传入 `"style": "invalid_style"`
- **预期结果**: `{"code": 40002, "message": "风格参数无效"}`
- **实际结果**: `{"code": 50001, "message": "服务器内部错误"}`
- **原因推测**: 参数校验在AI调用之后执行，或者校验失败时抛出了未被正确捕获的异常

### BUG-002：缺少style字段返回错误码不正确

- **严重程度**: 一般
- **接口**: `POST /api/v1/generate`
- **复现步骤**: 请求体仅含 `{"scene": "测试"}`，不含style
- **预期结果**: `{"code": 40004, "message": "style: 风格不能为空"}`
- **实际结果**: `{"code": 50001, "message": "服务器内部错误"}`

### BUG-003：场景超过200字符未返回40001

- **严重程度**: 一般
- **接口**: `POST /api/v1/generate`
- **复现步骤**: scene传入201个字符
- **预期结果**: `{"code": 40001, "message": "场景输入为空或超长"}`
- **实际结果**: `{"code": 50001, "message": "服务器内部错误"}`
- **注意**: 空字符串校验正常返回了40004，仅超长校验有问题

### BUG-004：GET请求POST接口返回50001

- **严重程度**: 轻微
- **接口**: `POST /api/v1/generate`
- **复现步骤**: 使用GET方法访问 `/api/v1/generate`
- **预期结果**: `405 Method Not Allowed`
- **实际结果**: `{"code": 50001, "message": "服务器内部错误"}`

### BUG-005：非JSON请求体返回50001

- **严重程度**: 轻微
- **接口**: `POST /api/v1/generate`
- **复现步骤**: 请求体传入纯文本 `"not json"`
- **预期结果**: `{"code": 40004, "message": "请求格式错误"}`
- **实际结果**: `{"code": 50001, "message": "服务器内部错误"}`

## 6. 测试结论

### 6.1 通过项

- **设备隔离**: 所有接口的 device_id 归属校验正确（40005），不同设备间数据完全隔离
- **收藏切换**: 幂等性正确，连续调用正确切换 true/false
- **级联删除**: 删除历史记录同步清理收藏数据
- **分页容错**: 负数页码等异常参数有容错处理
- **输入过滤**: 空值、缺失请求头等前端常见错误有明确的错误提示
- **敏感词过滤**: XSS输入被正确拦截，返回兜底文案

### 6.2 待修复项

1. 参数校验错误码需修正（BUG-001、002、003）— 统一返回 50001 不利于前端精确提示
2. 异常请求格式/方法的错误码需修正（BUG-004、005）
3. 建议增加 Controller 层 `@ExceptionHandler` 或全局异常处理中细化 HttpMessageNotReadableException（非JSON）和 HttpRequestMethodNotSupportedException（方法错误）的错误码

### 6.3 建议

- 当前 `@Valid` 校验可能未覆盖 style 枚举校验和 scene 长度校验，建议在 GenerateRequest DTO 中添加 `@Size(min=1, max=200)` 和自定义 `@ValidStyle` 注解
- 全局异常处理中建议将 `MethodArgumentNotValidException` 和 `HttpMessageNotReadableException` 映射为400系列错误码
- 建议在测试环境配置有效的 DeepSeek API Key，以便完整验证生成流程
